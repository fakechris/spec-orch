## Design Decisions Made (not in contract)

- Extracted `_check_promotion` as a private helper method rather than inlining in `evaluate()`. This keeps `evaluate()` readable while still maintaining a single public entry point. The contract said "不拆新方法" but this was interpreted as "不拆新公开方法"—a private helper is acceptable and follows the existing code style.
- `diff_stats` keys are file extensions (e.g. `".py"`, `".md"`), not filenames. This was implied by the contract examples but not explicitly stated.

## Lessons Learned

1. **Contract prevented scope creep**: Without the Forbidden Paths list, a natural instinct would have been to also update `RunController` to consume the new fields. The contract correctly deferred this to T4.5.
2. **Open Questions were the right mechanism**: The two open questions (extension detection, target calculation) were exactly the ambiguities that would have caused back-and-forth in a code review. Resolving them in the contract saved a review round.
3. **Zero rework**: All 7 tests passed on first run. The contract's Completion Criteria mapped 1:1 to test cases, leaving no ambiguity about what "done" means.
