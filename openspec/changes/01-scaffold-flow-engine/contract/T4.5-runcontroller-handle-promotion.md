# Task Contract: T4.5 — RunController 处理 GateVerdict.promotion_required

> Change: 01-scaffold-flow-engine
> Task ID: T4.5
> 前置: T3.3 (GateService promotion_required), T4.4 (步骤推进基于 FlowEngine)
> Spec 对应: REQ-3.3, REQ-7, S3.2, S7.1, S7.2

---

## 1. Intent

使 `RunController` 在收到 `GateVerdict.promotion_required=True` 时，终止当前流程，
切换到 `promotion_target` 指定的流程，并从新流程的起点重新开始。同时记录
`FlowTransitionEvent`。

**外部可观测结果**：
- Standard 流程中 Gate 返回 `promotion_required=True, promotion_target="full"` 时，
  RunController 切换到 Full 流程并从其第一个步骤开始
- 切换过程中产生 `FlowTransitionEvent` 被记录（通过 `record_flow_transition` 接口）
- 无效的 promotion（如 target 不存在、target 是当前或更低流程）被安全处理

---

## 2. Decisions

### 已确定

| 决定 | 理由 |
|------|------|
| promotion 检查放在 Gate evaluate 返回后、advance 决策前 | 自然的检查点，Gate 产出信号后 RunController 立即响应 |
| 切换后从新流程 FlowGraph 的 `steps[0]` 重新开始 | Spec REQ-3.3 明确要求"从起点重新开始" |
| `promotion_target` 是 `str`，RunController 通过 `FlowType(target)` 转换 | T3.3 已确定 Gate 产出 str，RunController 负责转枚举 |
| 无效 target 时记录错误并继续当前流程 | REQ-7.1 要求不静默失败但也不崩溃 |
| 调用 `record_flow_transition(event)` 记录切换事件 | 依赖 T5.1 的 stub 实现即可 |
| promotion 最多执行一次（不递归：切换后不再检查新流程的 promotion） | 避免无限循环 |

### Open Questions

| 问题 | 影响 | 建议 |
|------|------|------|
| promotion 后是否重新执行已通过的步骤？ | 如果 Standard 走到 Gate 才 promote 到 Full，Full 的 discuss/freeze/plan 是否执行？ | 建议按 Spec "从起点开始"，即全部重来。实际执行中 discuss/freeze 步骤如果 run_state=None 会被跳过 |
| 记录方式：FlowTransitionEvent 写入 activity log 还是 Memory？ | 影响持久化策略 | 本 task 只调 `record_flow_transition`（T5.1 的 stub），具体存储是 T5.2 的职责 |

## 3. Boundaries

### Allowed Paths

- 修改 `src/spec_orch/services/run_controller.py`：
  - 在 Gate evaluate 返回后增加 promotion 检查逻辑
  - 切换 `current_flow` 和 `current_step`
  - 调用 `record_flow_transition(FlowTransitionEvent(...))`
  - 处理无效 target 的错误路径
- 新增或修改 `tests/unit/test_run_controller.py` 中的测试
- 导入 `FlowTransitionEvent` 和 `record_flow_transition`

### Forbidden Paths

- **不得修改** `GateService` 的任何代码（T3.x 已完成）
- **不得修改** `GateVerdict` 或 `GateInput` 的字段定义
- **不得修改** `FlowEngine`, `FlowGraph`, `FlowMapper` 的实现
- **不得实现** `record_flow_transition` 的持久化逻辑（T5.1/T5.2 的职责）
- **不得实现** demotion 逻辑（T4.6 的职责）
- **不得实现** backtrack 逻辑（T4.7 的职责）
- **不得改变** `RunResult` 的字段结构
- **不得引入** 对 Memory 子系统的直接依赖

### Public Behavior That Must Not Regress

| 行为 | 验证方式 |
|------|---------|
| `run_issue(id)` 无 promotion 时行为不变 | 现有测试 |
| `GateVerdict.promotion_required=False` 时不触发任何切换 | 现有测试 + 新测试 |
| `advance()` 行为不变 | 现有测试 |
| `pytest --tb=short` 全量通过 | 全量回归 |
